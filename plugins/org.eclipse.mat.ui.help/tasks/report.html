Problems displaying reports

Sometimes Memory Analyzer does not display reports such as Leak Suspects report or Component report and displays a message such as: Failed to create a browser because of: org.eclipse.swt.SWTException: Failed to execute runnable (org.eclipse.swt.SWTError: No more handles because no underlying browser available. or org.eclipse.swt.SWTException: Failed to execute runnable (org.eclipse.swt.SWTError: XPCOM error -2147467259)

  1. Close Memory Analyzer.
  2. Note:

    On a Linux system, make sure that libwebkit2gtk-4.0 is installed.

    Select the latest version available of libwebkit2gtk-4.0 and install it.
    • On an Ubuntu® system use commands such as 
      sudo apt-get update
sudo apt-get install libwebkit2gtk-4.0-37

    • On a Red Hat® system use rpm

    The operating system should confirm that libwebkit2gtk-4 has been properly installed

  3. Restart Memory Analyzer
  4. Open a heap dump.
  5. Run a report such as Leak Suspects report or Component report and check how the report looks.
    Check the HTML report is displayed.
